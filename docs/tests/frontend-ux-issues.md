### UI-001: Tooltip 在小屏设备上可能被遮挡

| 项目 | 内容 |
|------|------|
| **优先级** | P2 |
| **发现日期** | 2026-03-28 |
| **测试环境** | Chrome 最新版，宽度 768px |
| **问题描述** | 当指标卡片位于屏幕右边缘时，tooltip 向右弹出可能被屏幕边缘截断 |
| **复现步骤** | 1. 调整浏览器宽度到 768px<br>2. 找到最右侧的指标卡片<br>3. 悬停在问号图标上 |
| **预期行为** | tooltip 应自动调整方向，向左弹出 |
| **当前行为** | tooltip 被右边缘截断，内容显示不全 |
| **建议修复** | 添加检测逻辑，当右侧空间不足时改为向左弹出<br>可使用 `position: absolute; right: 100%;` |
| **状态** | ☐ 待修复 ☐ 修复中 ☐ 已修复 |

|------|------|
| **优先级** | P2 |
| **发现日期** | 2026-03-28 |

### UI-004: Toast 通知堆叠问题

| 项目 | 内容 |
|------|------|
| **优先级** | P2 |
| **发现日期** | 2026-03-28 |
| **测试环境** | Chrome 最新版 |
| **问题描述** | 快速连续触发多个操作时，Toast 通知可能堆叠在一起 |
| **复现步骤** | 1. 快速连续点击多个指标卡片<br>2. 观察 Toast 显示 |
