## 三、架构改进

### 3.1 建议方案

#### 3.1.1 依赖注入模式 (P0)

**目标**: 降低模块耦合，提升可测试性

**实施步骤**:
1. 定义 Provider 接口类型
2. 修改工厂函数接受依赖
3. 更新现有调用代码
4. 添加 mock 实现用于测试

**预计工作量**: 2-3 天

#### 3.1.2 统一错误处理 (P0)

**目标**: 类型安全的错误处理

**实施步骤**:
1. 定义 QuantError 类型层次
2. 更新模块返回类型
3. 添加错误转换辅助函数
4. 更新测试代码

**预计工作量**: 3-4 天

#### 3.1.3 事件驱动架构 (P1)

**目标**: 支持实时监控和通知

**实施步骤**:
1. 定义 Event 类型和 EventBus
2. 实现订阅/发布机制
3. 集成到回测引擎
4. 添加事件处理器示例

**预计工作量**: 4-5 天

#### 3.1.4 策略配置 DSL (P1)

**目标**: 提升策略定义可读性

**实施步骤**:
1. 定义 StrategyBuilder 类型
2. 实现流式 API
3. 更新内置策略使用 DSL
4. 添加策略示例

**预计工作量**: 3-4 天

#### 3.1.5 缓存层 (P1)

**目标**: 优化指标重复计算

**实施步骤**:
1. 定义 CachedIndicator 类型
2. 实现缓存失效机制
3. 集成到指标计算
4. 添加缓存统计

**预计工作量**: 4-5 天
