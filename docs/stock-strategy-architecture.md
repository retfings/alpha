## 系统概述

股票策略系统是一个基于 MoonBit 的量化交易回测平台，提供股票筛选、策略配置和回测分析功能。系统采用前后端分离架构，支持 CLI 和 Web 两种交互方式。

### 核心功能

| 功能模块 | 描述 |
|----------|------|
| 择股设置 | 股票池配置、基础筛选、选股指标配置 |
| 交易模型 | 策略定义、信号生成、回测执行 |
| 回测分析 | 绩效统计、收益曲线、回撤分析 |
| 风控管理 | 最大回撤、仓位限制、止损规则 |

```

   ↓

   ↓
extract_stock_code() / extract_backtest_id() ──→ 提取参数
   ↓
route_request() ──→ 路由到对应 Handler
   ↓
Handler 函数 ──→ 业务逻辑处理
   ↓
make_response() ──→ 创建 HTTP 响应
   ↓
HTTP 响应返回客户端
```

### 回测数据流

```
1. 用户配置回测参数
   ↓
2. 加载 K 线数据 (load_klines_from_csv)
   ↓
3. 计算技术指标 (indicator::*)
   ↓
4. 创建策略实例 (Strategy)
   ↓
5. 配置回测引擎 (BacktestEngine)
   ↓
6. 逐条处理 K 线
   ├── 策略生成信号 (on_bar)
   ├── 风控检查 (RiskEngine.check)
   ├── 执行交易 (Portfolio.buy/sell)
   └── 更新权益曲线
   ↓
7. 生成回测报告 (BacktestResult)
   ↓
8. 返回 JSON 响应 / 渲染图表
```

## 附录

### A. 端口配置

| 服务 | 默认端口 | 配置方式 |
|------|----------|----------|
| HTTP API | 8080 | `--port` 参数 |
| 静态文件 | 3000 | Python 服务器 |

### B. 数据格式

**股票代码格式**: `market.code`
- 上海股票：`sh.600000`
- 深圳股票：`sz.000001`

**日期格式**: `YYYY-MM-DD`

**响应格式**: JSON
```json
{
  "status": "success|error",
  "data": { ... },
  "error": {
    "code": "ERROR_CODE",
    "message": "..."
  }
}
```

### C. 错误码

| 错误码 | HTTP 状态 | 说明 |
|--------|----------|------|
| `INVALID_STOCK_CODE` | 400 | 无效的股票代码 |
| `DATA_NOT_FOUND` | 404 | 数据未找到 |
| `STRATEGY_NOT_FOUND` | 404 | 策略未找到 |
| `BACKTEST_FAILED` | 500 | 回测执行失败 |
